# Δαπάνη Ειδικών Αστυφυλάκων — Μηνιαία μεταφορά PDF → Excel

Εφαρμογή γραμμής εντολών (Python) που κάθε μήνα διαβάζει **οπτικά** τη σαρωμένη
επιστολή της Αστυνομίας Κύπρου προς τον ΟΚΥπΥ μέσω του Claude API (vision — όχι
tesseract/OCR, ώστε τα χειρόγραφα ✓ δίπλα στα ποσά να αγνοούνται σωστά),
επαληθεύει τα ποσά έναντι των συνόλων-ελέγχου της επιστολής και τα γράφει στο
υπάρχον workbook `ΧΡΕΩΣΕΙΣ_ΕΙΔΙΚΟΙ_ΑΣΤΥΦΥΛΑΚΕΣ_2026.xlsx`, στο σωστό φύλλο
μήνα, χωρίς να πειράξει φόρμουλες ή μορφοποίηση.

## Εγκατάσταση

```bash
python -m venv .venv && source .venv/bin/activate   # προαιρετικά
pip install -r requirements.txt
export ANTHROPIC_API_KEY=sk-ant-...                 # ποτέ hardcoded στο κώδικα
```

Απαιτείται Python 3.11+.

## Χρήση

```bash
python -m daspani "Επιστολή_Απρίλιος_2026.pdf" "ΧΡΕΩΣΕΙΣ_ΕΙΔΙΚΟΙ_ΑΣΤΥΦΥΛΑΚΕΣ_2026.xlsx"
```

Προαιρετικά:

| Επιλογή | Περιγραφή |
|---|---|
| `--minas ΑΠΡΙΛΙΟΣ` | Επιβολή φύλλου/μήνα μισθών. Αν λείπει, διαβάζεται από τον τίτλο της επιστολής. |
| `--model ...` | Μοντέλο Claude (προεπιλογή `claude-sonnet-4-6`, ή μεταβλητή `DASPANI_MODEL`). |
| `--dpi 230` | Ανάλυση απόδοσης σελίδων σε εικόνα. |
| `--yes` | Χωρίς ερωτήσεις επιβεβαίωσης (για scripts). |

## Τι κάνει βήμα-βήμα

1. **Rasterize** τις σελίδες με PyMuPDF σε ~230 dpi PNG. Οι σελίδες πίνακα
   (2 και μετά) περιστρέφονται +90°· αν είναι πολύ ψηλές, κόβονται σε πάνω/κάτω
   μισό με ~10% επικάλυψη και οι διπλές γραμμές αφαιρούνται (ΑΚΑ + ποσά).
2. **Σελίδα 1** → Claude API: τα 8 σύνολα-ελέγχου (Βασικοί, Τιμαριθμικό,
   Αύξηση 1,5%, Βάρδια, Κυριακή/Αργία, Εισφορά Εργοδότη, Διοικητικά 10%,
   Γενικό Σύνολο), μήνας μισθών/επιδομάτων και έτος.
3. **Σελίδες πίνακα** → Claude API: όλες οι γραμμές σε JSON (aka, east, name,
   date, basic, tim, auxisi, bardia, kyriaki, remarks), μαζί με τις γραμμές
   «ΑΝΑΛΟΓΙΑ …»/«ΕΠΙΔΟΜΑΤΑ …» και τυχόν αρνητικές διορθωτικές.
4. **Έλεγχος συνόλων**: άθροισμα ανά στήλη vs επιστολή, ανοχή ±0,05 €. Αν κάποια
   στήλη δεν κλείνει, γίνεται μία αυτόματη επανάληψη ανάγνωσης σε 300 dpi· ό,τι
   εξακολουθεί να αποκλίνει σημειώνεται για χειροκίνητο έλεγχο.
5. **Έλεγχοι συνέπειας** (μόνο πλήρεις μήνες): Αύξηση ≈ Βασικός×1,5% και
   Τιμάριθμος ≈ (Βασικός+Αύξηση)×12,67%. Οι αποκλίνουσες γραμμές σημειώνονται.
6. Το αποτέλεσμα του ελέγχου **εμφανίζεται πρώτα στον χρήστη** — η εγγραφή στο
   Excel γίνεται μόνο μετά από επιβεβαίωση.
7. **Εγγραφή** με openpyxl:
   - Ταίριασμα κάθε γραμμής μέσω **Α.Κ.Α.** (στήλη B). Γράφονται μόνο τα κελιά
     εισαγωγής G–K. Φόρμουλες L–P, μορφοποίηση, ΣΥΝΟΠΤΙΚΟ και το μπλοκ
     ΕΛΕΓΧΟΣ ΣΥΝΟΛΩΝ δεν αλλοιώνονται.
   - ΑΚΑ που δεν υπάρχει (νέα πρόσληψη) ή δεύτερη γραμμή ίδιου ΑΚΑ (αναλογία):
     νέα γραμμή πριν το ΣΥΝΟΛΟ, με αντιγραφή φορμουλών L–P από διπλανή γραμμή
     και επέκταση του εύρους του ΣΥΝΟΛΟΥ.
   - Τα σύνολα της επιστολής γράφονται στο μπλοκ «ΕΛΕΓΧΟΣ ΣΥΝΟΛΩΝ» (στήλη D
     του μπλοκ) και στο φύλλο ΣΤΟΙΧΕΙΑ_ΕΠΙΣΤΟΛΩΝ (μόνο αν ο μήνας δεν έχει
     ήδη τιμές).
   - Αποθήκευση ως **νέο αρχείο** `<όνομα>_<ΜΗΝΑΣ>.xlsx` δίπλα στο αρχικό —
     ποτέ overwrite χωρίς επιβεβαίωση.
8. **Τελική αναφορά**: πόσες γραμμές διαβάστηκαν, ποιες στήλες κλείνουν
   (OK/διαφορά), ποια ΑΚΑ δεν ταίριαξαν, ποιες γραμμές θέλουν χειροκίνητο έλεγχο.

Δεύτερο τρέξιμο με το ίδιο PDF δεν διπλογράφει γραμμές (idempotent): οι
υπάρχουσες γραμμές απλώς ξαναγράφονται με τις ίδιες τιμές.

## Δομή κώδικα

```
daspani/
  __main__.py       CLI και κύρια ροή
  pdf_images.py     PDF → PNG (PyMuPDF), περιστροφή, τεμαχισμός με επικάλυψη
  claude_extract.py Κλήσεις Claude API (vision), parsing JSON, συγχώνευση
  models.py         Δομές δεδομένων, parsing ποσών/μηνών
  verify.py         Έλεγχοι συνόλων και συνέπειας
  excel_write.py    Εγγραφή στο workbook (openpyxl)
  report.py         Εμφάνιση ελέγχου και τελικής αναφοράς
tests/              Unit tests (pytest) για parsing, ελέγχους και εγγραφή Excel
```

## Προσαρμογή στη δομή του workbook

Αν η διάταξη του workbook διαφέρει, οι σχετικές σταθερές είναι στην κορυφή του
`daspani/excel_write.py`:

- `FIRST_DATA_ROW`, `AKA_COL`, `NAME_COL`, `INPUT_COLS` (G–K), `FORMULA_COLS` (L–P)
- `CONTROL_VALUE_OFFSET`: η στήλη τιμών του μπλοκ ΕΛΕΓΧΟΣ ΣΥΝΟΛΩΝ
  (προεπιλογή: 3 στήλες δεξιά από την ετικέτα = στήλη D του μπλοκ)
- `STOIXEIA_SHEET`: όνομα του φύλλου με τα στοιχεία επιστολών

Τα ονόματα φύλλων δεν μετονομάζονται ποτέ (θα χαλούσαν οι αναφορές SUMIFS).

## Σημειώσεις

- Κόστος ανά μήνα: 2–3 εικόνες ανά τρέξιμο ≈ λίγα cents.
- Τα API calls γίνονται με temperature 0 και αυτόματο retry με backoff
  (έως 4 επαναλήψεις σε σφάλματα δικτύου/φόρτου).
- Αν προστεθούν νέες γραμμές πριν το ΣΥΝΟΛΟ, ελέγξτε ότι τα εύρη των SUMIFS
  στο ΣΥΝΟΠΤΙΚΟ καλύπτουν και τις νέες γραμμές (η εφαρμογή το υπενθυμίζει
  στην αναφορά).
