# Επίπτωση Υπέρβασης Εξειδικευμένων Μονάδων (ΟΚΥπΥ)

Μηνιαία εφαρμογή ενός αρχείου HTML: **`specialized-ceiling-app.html`** — το τελικό
παραδοτέο. Ανοίγει τοπικά σε οποιονδήποτε φυλλομετρητή, χωρίς εγκατάσταση, χωρίς
διακομιστή, πλήρως εκτός σύνδεσης (οι βιβλιοθήκες SheetJS και ExcelJS είναι
ενσωματωμένες μέσα στο αρχείο). Κανένα δεδομένο δεν φεύγει από τον υπολογιστή.

## Μηνιαία ροή εργασίας

1. Ανοίξτε το `specialized-ceiling-app.html` στον φυλλομετρητή.
2. Δώστε τα δεδομένα των μηνών με έναν από τους δύο τρόπους (ή συνδυασμό):
   - **Μεταφορά προηγούμενης περιόδου** (συνιστάται): σύρετε στη δική της ζώνη το
     `Επίπτωση_Υπέρβασης_….xlsx` που κατεβάσατε τον προηγούμενο μήνα — μεταφέρει
     όλους τους μήνες του (από το φύλλο «Δεδομένα») και προσυμπληρώνει τις
     παραδοχές (από το φύλλο «Εισαγωγές») — και μετά μόνο τα αρχεία του
     τρέχοντος μήνα.
   - **Όλα φρέσκα**: σύρετε τα IS Auditor Reports όλων των μηνών του έτους.
   Αν ένας μήνας υπάρχει και στη μεταφορά και ως νέο αρχείο, υπερισχύει το νέο
   αρχείο (εμφανίζεται σχετική ενημέρωση).
3. Σύρετε τα διαθέσιμα αρχεία **Conso >15%** και, προαιρετικά, το αρχείο
   **εκπτώσεων ΟΑΥ** («Ενημέρωση Νοσηλευτηρίων για αποζημίωση μονάδων υπέρβασης»).
4. Επιβεβαιώστε/διορθώστε τον μήνα κάθε αρχείου από το dropdown δίπλα του.
5. Ελέγξτε τα νούμερα στην ενότητα «Επισκόπηση» έναντι των αρχείων πηγής
   (οι μεταφερόμενοι μήνες σημαίνονται με «↻ από εξαγωγή προηγούμενης περιόδου»).
6. Πατήστε «Λήψη Excel» → `Επίπτωση_Υπέρβασης_Εξειδικευμένων_<έτος>_<μήνας>.xlsx`
   με 4 φύλλα (Σύνοψη · Υπολογισμός · Δεδομένα · Εισαγωγές) και **ζωντανές φόρμουλες**
   (μόνο SUMIFS/INDEX/MATCH/MAX/IF/SUM) — αλλάζοντας τις παραδοχές στο φύλλο
   «Εισαγωγές» μέσα στο Excel, όλα επανυπολογίζονται. Το αρχείο αυτό είναι και η
   «μεταφορά» του επόμενου μήνα — η αλυσίδα συνεχίζεται όλο το έτος.

Οι παραδοχές (συμφωνημένες μονάδες, βασικές τιμές, εκπτώσεις, διακόπτης πιστωτικών)
αποθηκεύονται στο localStorage του φυλλομετρητή· «Επαναφορά προεπιλογών» τις
επαναφέρει στις τιμές 2026.

## Λογική υπολογισμού

```
Προσμετρώμενες = Θετικές εξειδικευμένες − ΤΑΕΠ παραπομπές + ΤΑΕΠ >15% (Conso)
                 + ΑΝ(διακόπτης=ΝΑΙ)(Αρνητικές − Αρνητικές ΤΑΕΠ)
Συμφωνημένες μηνιαίες = ετήσιες ÷ 12
Υπέρβαση  = MAX(0, Προσμετρώμενες − Συμφωνημένες μηνιαίες)
Επίπτωση € = Υπέρβαση × Βασική τιμή × Τελικό % Έκπτωσης ΟΑΥ   (αρνητικό = απώλεια)
```

## Ανάπτυξη (developer notes)

```
npm install          # εγκαθιστά xlsx, exceljs (+ playwright για e2e)
npm run build        # src/app.src.html + src/core.js + βιβλιοθήκες → specialized-ceiling-app.html
npm test             # φτιάχνει συνθετικά fixtures και τρέχει τους unit ελέγχους
npm run test:e2e     # οδηγεί το χτισμένο HTML σε Chromium (upload → επισκόπηση → λήψη)
npm run test:recalc  # αποδεικνύει ότι οι φόρμουλες είναι ζωντανές: αλλάζει παραδοχές
                     # στο εξαχθέν xlsx και επανυπολογίζει με LibreOffice headless
npm run test:all     # όλα τα παραπάνω
```

- `src/core.js` — όλη η λογική (ανίχνευση μήνα, parsing IS Auditor/Conso/εκπτώσεων,
  υπολογισμός, έλεγχοι εξαγωγής, δημιουργία workbook). Τρέχει αυτούσιο και σε Node
  (τα tests ασκούν τον ίδιο κώδικα που «τρέχει» και στο HTML) και στον φυλλομετρητή.
- `src/app.src.html` — το UI· περιέχει markers `<!--INJECT:...-->` που το `build.js`
  αντικαθιστά με τις inline βιβλιοθήκες και το core.
- `test/` — γεννήτρια συνθετικών αρχείων (`make-fixtures.js`), unit tests
  (`run-tests.js`), browser E2E (`browser-e2e.js`), έλεγχος επανυπολογισμού
  (`recalc-test.js`). Τα συνθετικά δεδομένα αναπαράγουν τα δείγματα-άγκυρες των
  προδιαγραφών (π.χ. F1054 Ιανουάριος: excess ≈ 313,38 → impact ≈ −613 χιλ. €).

Μετά από κάθε αλλαγή στα `src/`, τρέξτε `npm run build` και κάντε commit και το
ανανεωμένο `specialized-ceiling-app.html` (είναι το παραδοτέο).

## Σημείωση για τους ελέγχους αποδοχής

Τα ακριβή ποσά των προδιαγραφών (π.χ. −613.054 € για F1054 Ιανουάριο, γενικό σύνολο
−3.924.979 € για Ιαν–Μάι) προϋποθέτουν τα πραγματικά αρχεία πηγής HIO, τα οποία δεν
περιλαμβάνονται στο αποθετήριο (δεδομένα υγείας). Οι αυτόματοι έλεγχοι χρησιμοποιούν
συνθετικά αρχεία που αναπαράγουν τις δημοσιευμένες ενδιάμεσες τιμές των προδιαγραφών
(pos/pos_ae/over-15 %/excess) και επαληθεύουν την επίπτωση εντός της ανοχής
στρογγυλοποίησης (−613.067 € έναντι −613.054 € με τις στρογγυλεμένες στα 2 δεκαδικά
εισόδους). Με τα πραγματικά αρχεία η πρώτη εκτέλεση πρέπει να συγκριθεί με το
workbook αναφοράς.
